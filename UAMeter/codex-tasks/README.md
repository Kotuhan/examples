# Codex tasks workflow

## Структура задачі
- Папка: `{ML|iOS}-{id}-{short_description}` (наприклад, `ML-1-convert-yolo-to-xcode`).
- Файли всередині:
  - `task.md` — опис задачі та очікуваний результат.
  - `plan.md` — детальний план з кількома альтернативними шляхами.
  - `review.md` — підсумок після виконання (що зроблено, перевірки, ризики).

## Кроки роботи
1) **Опис** — заповнити `task.md` (контекст, вимоги, артефакти на виході).
2) **Планування** — у `plan.md` дати щонайменше два альтернативні варіанти виконання з чекбоксами.
3) **Ревʼю** — після виконання заповнити `review.md` (результати, тести, ризики/next steps).

## Правила ведення
- Кожна нова задача створюється як нова папка з цією структурою.
- Оновлення плану/статусу фіксуємо у відповідних файлах, не перетираючи контекст.
- Якщо зʼявляються нові артефакти (скрипти, моделі), посилатися на точні шляхи.
